# Sandbox 性能测试报告

基于 `main.py` 跑 **100 次** `Sandbox.create()` + `run_code()` 完整生命周期统计。

## 测试环境

| 项 | 值 |
|---|---|
| template | `code-code-vpc` |
| 沙箱 timeout | 3600s |
| run_code 超时 | 60s |
| 网关 | `E2B_DOMAIN=ap-shanghai.tencentags.com` (腾讯 AGS e2b 兼容网关) |
| 测试负载 | `print('hello python')` |

每次循环做的事：

1. `start = time.time()`
2. `sandbox = Sandbox.create(template="code-code-vpc", timeout=3600)`
3. `result = sandbox.run_code("print('hello python')", timeout=60)`
4. `elapsed = time.time() - start`，打印 `create / run_code / lifecycle`
5. `sandbox.kill()`

> 计时只覆盖 `create + run_code`，不含 `kill()`。

## 总体统计 (n=100, 全部成功)

| 指标 | min | p50 | p90 | p95 | p99 | max | mean | σ |
|---|---:|---:|---:|---:|---:|---:|---:|---:|
| Lifecycle | 0.20 | 0.35 | 0.86 | 0.93 | 1.02 | 1.02 | 0.44 | 0.24 |
| create | 0.10 | 0.13 | 0.37 | 0.38 | 0.48 | 0.48 | 0.18 | 0.10 |
| run_code | 0.09 | 0.25 | 0.51 | 0.55 | 0.62 | 0.62 | 0.26 | 0.16 |

所有 100 次 `stderr` 为空、`Error: None`。

## 核心结论

### 1. create 和 run_code 都是双峰分布

不是正态抖动，是离散的 **快/慢两态**。

**`create` 分布**：

| 区间 (s) | 次数 | 直方 |
|---|---:|---|
| [0.00, 0.15) | 65 | `################################` (快簇 — AGS 预热池命中 / 模板已驻留) |
| [0.15, 0.20) | 11 | `#####` |
| [0.20, 0.30) | 4 | `##` |
| [0.30, 0.40) | 16 | `########` (慢簇 — 实际冷启动) |
| [0.40, 0.50) | 4 | `##` |

**`run_code` 分布**：

| 区间 (s) | 次数 | 直方 |
|---|---:|---|
| [0.00, 0.15) | 45 | `######################` (快) |
| [0.15, 0.25) | 5 | `##` |
| [0.25, 0.35) | 17 | `########` |
| [0.35, 0.45) | 16 | `########` |
| [0.45, 0.55) | 11 | `#####` |
| [0.55, 1.00) | 6 | `###` (慢) |

`run_code` 双峰更明显、整体更靠右（慢簇占比 40%）。

### 2. create ↔ run_code 强正相关: 0.720

意味着同一次 `Sandbox.create()` 的 **"冷/热"属性会沿用** 到后续 `run_code`：

| 状态 | create | run_code | 占比 |
|---|---|---|---:|
| AGS 预热池命中 / 模板已驻留 | ~0.10–0.13s | ~0.10–0.13s | ~48% |
| 需要真去调度（冷启动） | ~0.30–0.48s | ~0.30–0.60s | ~52% |

## 关键结论与建议

- **p50 = 0.35s，p95 = 0.93s**，典型值 1 秒内，最差接近 1 秒出头。
- **52% 的请求走冷启动路径**：客户端加机器无法直接缓解，瓶颈在 AGS 网关侧模板预热池规模不足。
- **客户端已经没有继续压缩的空间**：0.10s 量级已经接近网络 RTT。
- 优化方向应在 **网关侧**：
  - 让 `code-code-vpc` 模板预热更多实例
  - 或让 `Sandbox.create()` 更激进地复用已存在的 sandbox
- 若用于交互式产品，SLA 应按 **p95（不是 p50）** 对外承诺：p50 是 0.35s "很快"，但 p95 才是真实体感。

## 附：同步/异步

`Sandbox.create()` 是 **同步**调用。`e2b_code_interpreter.Sandbox` 继承自 `e2b.sandbox_sync.Sandbox`，整条调用链（`requests` / HTTPX 同步客户端）会阻塞当前线程，直到远端真正把沙箱起好。

100 次串行累计无法用多线程压下来，除非：

- 切换到 `AsyncSandbox`（`asyncio`），或
- 多进程 / 多线程并行触发多个 sandbox。
